1. 系统初始化过程简略总览

    如上图所示，当 Kernel 被引导器拷贝到内存中，并且掌握 CPU 的控制权时，它会首先开始执行 start_kernel 函数，这就是 Kernel 的入口，它是在 init/main.c 中进行定义的。它的主要功能是完成 Kernel 在投入运行之前所有的初始化工作，其主要工作可以被整理为：

  • Boot-time Options: 两次对 parse_args 的调用 (i.e. 一次直接调用一次间接调用) 对来自 boot loader (e.g. LILO, GRUB) 传递的配置参数进行了处理;
  • Interrupts and Timers: 软件中断和硬件中断分别被 softirq_init 和 init_IRQ 进行了初始化; 定时器资源在此时也会被进行初始化;
  • Initialization Routines: 内核子系统和内置的设备驱动被 do_initcalls 进行了初始化; free_init_mem 将一些不需要使用到的代码段占用的内存空间进行了释放; run_init_process 启动了整个系统的第一个进程，通常来说是 init 进程，其 PID 为 1;

    在 start_kernel 函数结束之前，它会调用 kernel_init 内核线程来完成剩余的初始化工作。我们在本章中看到的很多初始化工作就是在 kernel_init 内核线程中的 do_basic_setup 中完成的。

    网络设备初始化的主要工作是：

  1. 中断线 (Interrupt Line) 初始化
  2. I/O ports 和 I/O Memory 的初始化
