1. MVC 背后的思想

    MVC 是一种应用程序的开发模型，由于我们在本系列讨论的是 Web 应用的开发，所以我们把描述 MVC 的范围缩小至 Web应用的一种开发模型。常见的 Web 开发框架或多或少都会有 MVC 模型的影子。MVC 开发模型将应用程序的模块按照功能逻辑区分开，由 模型(Model)，视图(View) 和 控制器(Controller) 三大部分组成。如上图所示，以下我们将整个对 Web客户端 的请求的处理流程串起来，分别对这三者的功能进行阐述。

    当 Web客户端 向 Web服务端 发送一个请求时，首先会有一个 Router 来根据请求内容选择一个对应的 Controller 进行处理。Controller 接收到请求后，会从 View 中获取与请求相关的信息，如果在处理过程中有需要的话，它还会向 Model 请求数据。Model 是负责与数据库进行交互，向 Controller 响应其请求的数据的模块。当 Controller 处理完数据后，其会重新更新 View。这个更新过程大多数情况下是使用 Template Engine (模版引擎) 来实现的，Template Engine 用于创建动态网页内容，以满足各种网页显示需求。当响应的 View 创建完成后，Controller 就会将更新过的 View 响应给 Web客户端。

2. MVC 的伪代码 (助于理解一些 Web Framework)

    为了更好的理解 Web Framework，我们给出上面这段伪代码，以方便我们结合对 MVC 处理流程来进行理解。下面我们同样按照对 Web 客户端请求的响应流程，对这段伪码进行分析。
    首先，Web 客户端 会对一个特定的 URL 发起请求，在收到这个请求后，根据 URL 指示的资源地址，Router 会选择一个对应的 Controller 进行调用，在上面的例子中我们调用了 User 类的 getProfile(id) 方法，并且给这个 Controller 传递了相关参数。
    Controller 在获取到相关参数后，在内部他又会去调用 Model 的方法，在我们的例子中 getProfile(id) 方法 内部又调用 UserModel 类的 getProfile(id) 方法。在 UserModel 类的 getProfile(id) 方法中，我们看到它使用了 SQL 语句去向数据库发起查询，并将查询结果返还给 Controller，也就是例子里的 User 类。
    User 类拿到这个返回之后就会去更新 View，在我们的例子中就是语句 renderView('users/profile', profile)。在上面伪码的最下方我们看见 Views 渲染页面的时候留了许多用于动态渲染的地方，这些地方就是用来接收来自 Controller 传递过来的参数，从而实现对不同的查询请求渲染出不同的网页的功能。
