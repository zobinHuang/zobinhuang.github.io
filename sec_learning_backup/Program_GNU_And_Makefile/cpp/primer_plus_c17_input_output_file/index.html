1. 背景

    与BASIC，Pascal等语言不同，C/C++并没有把输入和输出作为语言的一部分（即并没有与输入、输出直接相关的关键字）。C语言把I/O的设计留给了编译器的开发人员，以便让实现人员能够更自由地实现I/O函数。在实际情况中，大多数编译器的实现人员都把输入、输出的实现做在了最初为UNIX环境开发的库函数的基础之上。ANSI C（美国国家标准协会 C标准）在承认这个软件包时称其为 “Standard Input/Output”，这也就是 “stdio.h” 的由来。
    而在C++中，ANSI/ISO C++把 “iostream” 和 “fstream” 这两个包含了若干类的文件作为了一个标准类库，以提供C++的I/O解决方案。以下对其展开讨论。

2. 流(stream)和缓冲区(buffer)的概念

(1) 流

    C++把输入和输出看作字节流。输出流中的字节可能来自键盘、存储设备或其它程序，同样地输出流中的字节可能前往屏幕、打印机、存储设备或其他程序。因此，当我们在使用输入/输出流的时候，我们必须指明流的两个端点分别是哪里。如上图所示，对于输入流，文件端部提供了流的来源，程序端部的连接将流的输出字节转储到程序中。同样地，对于输出流也需要将输出流连接到程序，以及将流与输出目标关联起来。

(2) 缓冲区

    光有流的概念还不够，想象一下，磁盘驱动器这样的设备以512字节的块为单位来传输信息，而程序通常每次只能处理一个字节的信息。因此需要有 缓冲区 来匹配流两边不同的带宽。在输入时，程序实际上会从磁盘上读取大量信息至缓冲区中，然后每次从缓冲区中缓慢地读取字节，在内存中的数据读取与在磁盘中相比，会快很多。同样地，在输出时，程序实际上会把输出的字节写到缓冲区中，然后把整块缓冲数据区传输给硬盘，然后清空缓冲区，以上输入/输出流对缓冲区的操作被称为 刷新缓冲区 (flushing the buffer)。
    值得一提的是，对于源自键盘的输入流，由于键盘的输入速率本身就很缓慢，因此其实在这种情况下没有必要设置缓冲区，但为了使得用户可以在将输入数据传输给程序之前修改输入数据，因此C++对键盘输入流的处理是等到键入回车之后才刷新缓冲区。
