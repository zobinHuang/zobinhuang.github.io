    Section 1. Docker 帮助命令
        docker version：显示当前 Docker 版本；
        docker info：显示 Docker 的系统信息，包括容器和镜像的数量等
        docker --help：帮助命令
        docker stats：显示各个容器消耗宿主机 CPU 和内存等的情况

    Section 2. Docker 镜像命令
        docker images：查看所有本地镜像
        docker search：搜索镜像 (from "https://hub.docker.com/")
        docker pull：下载镜像
        docker rmi：根据镜像 ID 来删除镜像

    Section 3. Docker 容器命令
        docker run [可选参数] 镜像名：根据一个镜像启动一个容器
        docker ps：显示当前正在运行的容器
        exit / 「Ctrl+P+Q」：退出容器
        docker rm：删除指定容器 ID 的容器
        docker start/restart/stop/kill 容器ID：启动/重启/停止/强制停止一个容器
        docker commit：提交自定义容器

    Section 4. Docker 其它常用
        docker run -d [镜像名]：以后台的方式运行容器
        docker logs [可选参数] 容器ID：查看 Docker 日志
        docker top 容器ID：查看容器中的进程信息
        docker inspect 容器ID：查看容器元数据
        docker exec -it 容器ID bashShell：进入当前正在运行的容器中 (创建一个新的终端)
        docker attach 容器ID：进入当前正在运行的容器中 (不创建一个新的终端)
        docker cp 容器ID:容器内路径 目的主机路径：从容器中拷贝文件到主机上

2. Docker 镜像命令

docker images

    查看所有本地镜像

  • REPOSITORY: 拉取仓库的名称
  • TAG: 镜像版本标签
  • IMAGE ID: 镜像 ID
  • CREATED: 镜像创建时间
  • SIZE: 镜像大小

    可选项 (可组合)：

  • -a: 列出所有镜像
  • -q: 只显示镜像 id

docker search

3. Docker 容器命令

docker run [可选参数] 镜像名

    根据一个镜像启动一个容器，常用可选参数如下：

  • --name="Name": 给容器起个名字，用于区分容器
  • -d: 以后台方式运行
  • -it: 使用交互方式运行，可以进入容器内部进行操作
  • -it --rm: 容器退出交互后立即删除容器
  • -e: 设置启动参数，根据不同的镜像有不同的配置参数
  • -v: 挂载数据卷，方法有
        -v 主机目录:容器内目录 (指定目录挂载)
        -v 容器内目录 (匿名挂载)
        -v 数据卷名称:容器内目录 (具名挂载)
        -v xxx:ro / xxx:rw：给挂载的目录增加权限 (ro: 只读，rw: 可读写(默认))
  • -p: 指定端口，方法有：
        -p ip:主机端口:容器宽口
        -p 主机端口:容器宽口【常用】
        -p 容器宽口
        容器宽口
  • -P: 随机端口指派

docker ps

    显示当前正在运行的容器，常用可选参数有：

  • -a: 查看当前和曾经运行过的容器
  • -n=?: 显示最近 n 个创建过的容器
  • -q: 仅显示容器 ID

exit / 「Ctrl+P+Q」

    退出容器，区别如下：

  • exit: 退出并停止容器
  • 「Ctrl+P+Q」: 退出但不停止容器

docker rm [容器 ID]

    删除指定容器 ID 的容器，但是不能删除正在运行中的容器。如果要强制删除，则使用 -f 可选参数。类似于我们在删除镜像中所展示的那样，我们可以通过下面的方式来删除所有的容器：


docker start/restart/stop/kill 容器ID

  • start: 启动一个已经停止的容器
  • restart: 重启一个正在运行的容器
  • stop: 停止一个正在运行的容器
  • kill: 强制停止当前容器 (在 docker stop 报错的情况下)
