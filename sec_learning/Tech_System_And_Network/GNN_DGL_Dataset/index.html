    DGLDataset 类有抽象函数 process， __getitem__ 和 __len__。子类必须实现这些函数，同时 DGL 也建议实现保存和导入函数，因为对于大型数据集，如果可以把处理后的数据集保存下来，那么将可以节省大量的时间。

    DGLDataset 的目的是提供一种标准且方便的方式来导入图数据。用户可以存储有关数据集的图、特征、标签、掩码，以及诸如类别数、标签数等基本信息。诸如采样、划分或特征归一化等操作建议在 DGLDataset 子类之外完成。

处理数据

    用户可以在 process 函数中实现数据处理，该函数假定原始数据已经位于 self.raw_dir 目录中。图上的机器学习任务通常有三种类型：整图分类、节点分类和链接预测。本节将展示如何处理与这些任务相关的数据集。

节点分类

    与整图分类不同，节点分类通常在单个图上进行。因此数据集的划分是在图的节点集上进行。 DGL 建议使用 节点掩码 来指定数据集的划分。下面我们以 citation_graph dgl_citation_graph 数据集为例展开分析。

对数据源文件进行分析

    首先我们从 数据源 下载到我们本节要使用的数据集 —— cora_v2。下载下来的数据集文件如下所示。

  • ind.cora_v2.x: 训练集节点的特征向量，类型为 scipy.sparse.csr.csr_matrix 实例;
  • ind.cora_v2.tx: 测试集节点的特征向量，类型亦为 scipy.sparse.csr.csr_matrix 实例;
  • ind.cora_v2.allx: 包含了部分 labeled 和 unlabeled 训练集节点的特征向量，是 ind.cora_v2.x 的超集，类型亦为 scipy.sparse.csr.csr_matrix 实例;
  • ind.cora_v2.y: 训练集节点的 one-hot 标签，类型为 numpy.ndarray 实例;
  • ind.cora_v2.ty: 测试集节点的 one-hot 标签，类型为 numpy.ndarray 实例;
  • ind.cora_v2.ally: ind.cora_v2.allx 数据集的标签，其中部分节点拥有标签，部分节点没有标签，类型为 numpy.ndarray;
  • ind.cora_v2.graph: 图的拓扑，以 {index: [index_of_neighbor_nodes]} 的形式存储为类型为 collections.defaultdict 的实例;
  • ind.cora_v2.test.index: 测试集节点的标号

    在上面的数据集文件中，allx (ally) 包含了两部分: x (y) 的被 labeled 的样本 (140 个)，以及其它的一些没有被 labeled 的样本 (1568 个)。另外，测试集 tx (ty) 的大小为 1000 个样本。

保存和加载数据

    DGL 建议用户实现保存和加载数据的函数，将处理后的数据缓存在本地磁盘中。这样在多数情况下可以帮用户节省大量的数据处理时间。DGL提供了 $4$ 个函数让任务变得简单。

  • dgl.save_graphs dgl_save_graphs 和 dgl.load_graphs dgl_load_graphs: 保存 DGLGraph 对象和标签到本地磁盘和从本地磁盘读取它们;
  • dgl.data.utils.save_info 和 dgl.data.utils.load_info: 将数据集的有用信息 (i.e. python dict 对象) 保存到本地磁盘和从本地磁盘读取它们。
