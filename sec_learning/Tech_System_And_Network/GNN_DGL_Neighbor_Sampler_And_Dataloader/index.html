Neighbor Sampler

DGL 设计

    邻居采样器的工作原理如 img_ns 所示，针对每一个目标节点的训练，在逐层的消息传递中，我们仅基于部分邻居进行信息的收集和更新，而不会使用完整的 $k$-跳邻居，以缩减每个 mini-batch 的规模，以及防止模型对训练集发生过拟合。

    img_ns 中展示了 DGL 中关于 Message Flow Graph (MFG, 消息流图) 的邻居采样设计思路。对于选定的若干根节点的 $k$-跳邻居，每一跳实际上都是一个二分图，DGL 把这样一个二分图称为一个 MFG。由于应用了邻居采样机制，因此每一个 MFG 中实际上只包含了真实节点的子集。在训练阶段，针对这些根节点的训练可以基于这些 MFGs 进行。
