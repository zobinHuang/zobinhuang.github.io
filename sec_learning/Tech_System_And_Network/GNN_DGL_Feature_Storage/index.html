特征存储

Frame: 存储某种类型节点/边的特征数据

    如上所示，Frame 本质上是一个字典，存储的是某种类型的点/边的若干种特征。它是在 dgl/python/dgl/frame.pydglsrc_python_dgl_frame_py 中被定义的，它的构造函数如下所示:

    可以发现在 Frame 中，私有变量 _columns 所存储的字典中，字典的 Key 代表的是特征的名称，而字典的 Value 代表的则是对应特征数据，并且特征数据是使用 Column 类实例进行存储的。实际上，特征数据是用底层使用的 Backend (e.g. PyTorch, Tensorflow, MXNet, etc.) 对应的 Tensor 进行存储的，Tensor 的第一维的长度即为该类型下点/边的个数，记录于 Frame 的私有变量 _num_rows 中，其余维度代表的即为具体的特征数据。

    下面我们对用于存储某种类型节点/边的某种特征数据的 Column 类进行分析。

    可以看到 HeteroNodeDataView 类是专门用于处理用于操作节点特征的类 (i.e. 当 G.ndata[ntype] 被调用时)。它的私有变量含义如下所示:

  • _graph: 存储着本次操作对应的 DGLHeteroGraph 实例;
  • _ntype: 一个列表，存储着本次操作关心的节点类型;
  • _ntid: 一个列表，存储着本次操作关心的节点类型对应的 index;
  • _nodes: 存储着本次操作关心的节点，若值为 ALL，则为全部节点;

    HeteroNodeDataView 类的 __getitem__ 方法用于返回用户要求的节点特征数据; __setitem__ 方法用于设置用户传入的节点特征数据；__delitem__ 方法用于删除用户指定的节点特征数据，它们分别调用了 DGLHeteroGraph 类的 _get_n_repr、_set_n_repr 和 _pop_n_repr 方法来实现相关的功能。 这里可以学到一种代码的写法，为了支持 instance.xdata['type'] 的用户 API 写法，我们可以采用上面所展示的 View 范式，通过在基础类 (i.e. instance) 的相关函数中返回出一个 'xxView' 类 (i.e. HeteroNodeDataView)，然后在 'xxView' 中利用基础类的相关函数 (i.e. _get_n_repr 等) 完成对数据的增删改查。
