前言

    在 Python 中，我们可以基于 thread 模块实现所谓 多线程 (Multithreading)，它使得程序的逻辑可以整理为多线程形式，然而基于 thread 实现的所谓多线程实际上是伪多线程。如 img_mt 所示，我们以 CPython 解释器为例，CPython 解释器引入了 Global Interpreter Lock (全局解释器锁，GIL) 的概念，简单说就是一个解释器实例在某一时刻当且仅当只能运行一条 Thread 中的逻辑，这使得实际上多线程的程序实际上是以串行的方式被执行的，因此被称为伪多线程。关于多线程和 GIL 的更多相关概念可以参考 cpython_gil。

    在 Python 中要实现真正的程序并行运行，我们需要基于 multiprocessing 模块实现 多进程 (Multiprocessing)，本质上就是通过 fork() 或者 spawn() 系统调用实现进程上下文的自我复制，如 img_mp 所示，通过在多个 CPU 核心上运行多个解释器，以承载多个进程，来实现真正的程序并行执行。

    在本文中，我们介绍 Python 中基于 multiprocessing 模块的多进程相关特性。

两种创建进程的模式 —— fork 和 spawn

    Python 多进程有两种创建进程的模式，spawn 与 fork，详细可见 fork_spawn。

    对于 fork 模式，主进程会直接自我复制给子进程运行，并把自己所有资源的 Handle 都让子进程继承，因而创建速度很快，但更占用内存资源。

    对于 spawn 模式，主进程只会把必要的资源的 Handle 交给子进程，因此创建速度稍慢。

共享内存

    为了基于 Python 实现多进程通信，上面提及的 Pipe 和 Queue 把需要通信的信息从内存里深拷贝了一份给其他线程使用，需要分发的线程越多，其占用的内存越多，而本节介绍的 Shared Memory (共享内存) 是一种不需要进行深度拷贝的多进程通信方式，其原理是解释器会负责维护一块共享内存，这块内存每个进程都能读取到，读写的时候遵守同步管理相关的规则。

    在 multiprocessing 模块中，基于 Manager 类可以创建一块共享的内存区域，但是存入其中的数据需要按照特定的格式；在 multiprocessing.sharedctypes 模块中，Value 类可以保存数值，Array 类可以保存数组。
