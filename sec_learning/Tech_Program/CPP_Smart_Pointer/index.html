shared_ptr

用法

基础用法

    上述的 std::auto_ptr 和 std::unique_ptr 都只能有一个智能指针指向某个对象，在 C++11 中，还有一个智能指针 std::shared_ptr，可以实现多个智能指针指向同一个对象的功能，并且是线程安全的。简单来说，std::shared_ptr 是基于引用计数实现的: 引用计数的原理是，多个 std::shared_ptr 智能指针同时引用一个对象，每当引用一次，引用计数加一；每当智能指针销毁了，引用计数就减一。当引用计数减少到 0 的时候就释放引用的对象。这种引用计数的增减发生在智能指针的构造函数，复制构造函数，赋值操作符，析构函数中。

std::weak_ptr

Motivation

    上面介绍的 std::shared_ptr 可以理解为在 raw pointer 的基础上 wrap 了一个 reference counter 的实现。因此每使用一个 std::shared_ptr 指向一个对象，对应的 reference count 就会被加 1。在这层机制的基础上，考虑两个 std::shared_ptr 之间相互引用的场景:

    下面我们对 main 函数的执行过程进行分析:

  1. Line 16 使得 p_parent 引用了一个 Parent 实例，该 Parent 实例的引用计数为 1; Line 17 使得 p_child 引用了一个 Child 实例，该 Child 实例的引用计数为 1;
  2. Line 18 调用了 std::shared_ptr< Child > 类的赋值操作符，使得该 Child 实例的引用计数变为 2; Line 19 调用了 std::shared_ptr< Parent > 类的赋值操作符，使得该 Parent 实例的引用计数变为 2;
  3. main 函数返回之前首先调用了 std::shared_ptr< Parent > 实例的析构函数，使得 Parent 实例的引用计数变为 1;
  4. main 函数返回之前随后调用了 std::shared_ptr< Child > 实例的析构函数，使得 Child 实例的引用计数也变为 1;

    可见，main 函数退出时，并没有把 Parent 实例和 Child 实例的引用计数变为 0，此时两个实例均不会被释放，这时候就造成了内存泄漏。出现泄漏的原因就是 p_parent 和 p_child 所引用的对象之间发生了相互引用，导致它们的引用计数在函数退出时不能减少到 0。

    如果把 Line 18 或者 Line 19 任意删除一行，则就不会有泄漏发生: 假设删去 Line 18，则 Child 实例的引用计数最高到 1，Parent 实例的引用计数最高到 2。在 main 函数退出时，假设先运行 std::shared_ptr< Parent > 实例的析构函数，则 Parent 实例的引用计数被减为 1，仍没有被释放；随后运行 std::shared_ptr< Child > 实例的析构函数，则 Child 实例的引用计数被减为 0，Child 实例的析构函数被运行，此时 Child 实例中包含的 std::shared_ptr< Parent > 也将被析构，此时 Parent 实例的引用计数被减为 0，Parent 实例的析构函数也被运行。因此，在 main 退出前，Parent 和 Child 实例均被释放，没有发生内存泄漏。

    现在问题已经明确: 使用强引用 std::shared_ptr 的情况下，在相互引用时将会造成资源无法被释放的问题，怎么解?

用法

    用于解决上述问题的方法是 std::weak_ptr，如 weak_ptr 所示，我们把 Parent 和 Child 之间互指的指针类型改为了 std::weak_ptr。使用 std::weak_ptr 并不会增加背后实际对象的引用计数，这说明背后实际对象可以在有 std::weak_ptr 正在指向它时被释放。因此从图中我们可以看见，即使 Parent 和 Child 之间发生了相互引用，在函数结束时它们的引用计数都变为了 0，因此会被释放。

    实际上，std::weak_ptr 指向的是 std::shared_ptr，换句话说，std::weak_ptr 是使用 std::shared_ptr 初始化的。std::weak_ptr 是 std::shared_ptr 的观察者，它不会干扰 std::shared_ptr 所共享对象的所有权。当一个 std::weak_ptr 所观察的 std::shared_ptr 要释放它的资源时，它会把相关的 std::weak_ptr 的指针设置为空，防止 std::weak_ptr 持有指向不明地址的指针。

    并且，很重要的是，std::weak_ptr 并不拥有背后实际对象的所有权，所以不能直接使用资源。std::weak_ptr 并没有重载 -> 和 * 操作符，所以不能通过他来直接使用资源，我们可以通过 std::weak_ptr 的 lock 方法来获得一个 std::shared_ptr 对象。示例代码如下所示:
