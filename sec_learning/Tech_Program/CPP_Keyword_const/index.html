    这里我们需要补充一下在实例化对象的过程中，使用 const 进行修饰的原因: 当我们使用 const 修饰对象实例化时，这说明该对象的成员变量在实例化之后就不能被改变。

    下表展示了一些对应关系:

const object
(成员变量不能改变)		 non-const object
(成员变量可以改变)
const 成员函数
(保证不修改成员变量)		 允许调用 允许调用
non-const 成员函数
(不保证不修改成员变量)		 不允许调用 允许调用

    另外，如果实在需要在 const 修饰的成员函数中修改成员变量，做法是使用 mutable 关键字来修饰该成员变量，这样一来它就可以被 const 修饰的成员函数所修改。

    最后需要补充的是，如果成员函数同时具有 const 和非 const 两个版本的话， const 对象只能调用 const 成员函数，而非 const 对象只能调用非 const 成员函数。
