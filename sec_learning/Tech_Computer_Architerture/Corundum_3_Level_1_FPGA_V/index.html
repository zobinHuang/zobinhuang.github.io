前言

请先按照 搭建 Corundum CR/仿真/综合/构建 环境 一文的说明，准备好环境。

Overview

    我们后续的说明将以 U250 板卡为例，其顶层文件将位于 fpga/mqnic/AU250/fpga_100g/rtl/fpga.vsrc_fpga_mqnic_au250_fpga_100g_rtl_fpga 文件下。该文件在最顶层梳理了 Corundum 的模块组成，其涵盖的功能如 corundum_top_level 所示:

    其中: 1 Clock & Reset Subsystem 描述了全局所有模块所使用的时钟信号的分配形式；2 Flash Subsystem 提供了 xxx 的支持；3 CMS Subsystem 提供了 xxx 的支持；4 PCIe Subsystem 借助 Xilinx 官方提供的 PCIe IP，提供了 NIC 和 Host 之间的交互能力；5 CMAC & GTY Subsystem 借助 Xilinx 官方提供的 IP —— UltraScale+ Integrated 100G Ethernet Subsystem xilinx_cmac_web，提供了芯片接入高速 Ethernet 的能力；6 DDR Subsystem 提供了 Corundum 访问板上 DRAM 资源的能力。下面我们分别对这几个 Subsystem 进行说明。

Clock & Reset Subsystem

    首先，顶层文件描述了 Corundum 是如何将时钟信号分配给各个模块的，本文梳理出来的结构如 corundum_clock_tree 所示，可以分为五部分:

  • 1
  • 2 的输入是 PCIe 的差分参考时钟，经过差分缓冲器 IBUFDS_GTE4xilinx_ibufds_gte4 后，输出的 O 和 ODIV2 两路时钟信号被送入 PCIe IP，后者会输出一路 250MHz 的 pcie_user_clk，被送入后续其它模块中；
  • 3 的输入是两对来自 QSFP 模块的差分时钟信号，经过差分缓冲器 IBUFDS_GTE4xilinx_ibufds_gte4 后，O 端信号用于驱动后面实例化出来的 CMAC 模块中的 GTY 收发器，而 ODIV2 信号经过 BUF_GT 原语后，得到一路 161MHz 的时钟信号，其一方面作为 ptp_clk 提供给核心模块 fpga_core，另一方面提供给 MMCME4_BASE 原语；
  • 4 基于上面得到的 161MHz 的时钟信号，其使用了
  • 5

    Reference Clock Input/Output Structure 是 GTY 收发器中负责参考时钟处理的部分，其有两种模式:

  1. Input: 从芯片时钟引脚上获取差分参考时钟信号，经处理后得到参考时钟信号，用于驱动 GTY 收发器的 Quad 或者 channel PLLs；
  2. Output: 将从一个 GTY Quad 中的任意一个 Channel 恢复出来的时钟信号，经处理后输出到芯片时钟引脚上，用于驱动其它器件；

    Input Mode 对应 IBUFDS_GTE3/4 原语；Output Mode 对应 OBUFDS_GTE3/4 原语。

IBUFDS_GTE3/4 原语

    IBUFDS_GTE3/4 是 Gigabit Transceiver 的差分输入时钟缓冲器，corundum_ibufds_gte4 可以看到，IBUFDS_GTE3/4 原语接收一对差分时钟输入，其输出有两个 csdn_ibufds_gte3 xilinx_gty_official_guide:

  • O 端为参考时钟输出，用于驱动 GTYE3/4_COMMON 或 GTYE3/4_CHANNE 原语的 GTREFCLK[0/1] 时钟信号；
  • ODIV2 端通过 REFCLK_HROW_CK_SEL 配置输出模式，默认 REFCLK_HROW_CK_SEL=2’b00，也即和 O 端保持相同的时钟信号，其可以通过 HROW routing 驱动 BUF_GT 原语 xilinx_bufg_gt，再由后者驱动其它模块
